## 背景・目的
召喚ボタンが押せる状態を検出し、1タップで召喚を実行するロジックを実装する。

## 要件

### やること
- `SummonAction` クラスの実装
  - `execute()` → 召喚ボタンをタップして召喚を実行する
- `ScreenAnalyzer` の `SUMMON_AVAILABLE` 状態検出をトリガーとする
- タップ後、画面が元の状態に戻るまで待機する（最大待機時間を `config.yaml` で設定）
- 召喚実行をログに記録する

### やらないこと
- 召喚するキャラクターの選択
- 召喚コストの管理

## 完了条件（DoD）
- [ ] `SUMMON_AVAILABLE` 検出時に `SummonAction.execute()` が呼ばれ、ボタンがタップされる
- [ ] タップ後に画面変化を待機してから次の処理へ進む
- [ ] 召喚実行がCLIログに「[SUMMON] 実行」として出力される
- [ ] タイムアウト時に警告ログが出てフリーズしない
